## 🎯 KEYWORD RESEARCH WORKFLOW

### Step 1: Find Your Keyword
```
Use Google Keyword Planner, Semrush, or Ahrefs
Look for keywords with:
- 1000+ monthly searches (sweet spot: 500-5000)
- Keyword Difficulty < 40 (easier to rank)
- High search intent (educational, commercial)
```

### Step 2: Analyze Competitors (SERP Analysis)
```
1. Google your target keyword
2. Check top 3 ranking pages
3. Record:
   - Their word count
   - Number of headings
   - Backlink count
   - Content structure
4. Plan: How can you create BETTER content?
   - More comprehensive?
   - Better examples?
   - More recent data?
   - Better organization?
```

### Step 3: Find Related Keywords
```
Use AnswerThePublic or AlsoAsked.com
Find:
- Questions users ask
- Related searches
- Long-tail keywords
Example: "study abroad" → "how to apply for study abroad", "cost of study abroad", etc.
```

## 🎓 BEST PRACTICES FOR SEO

### Keyword Strategy
```
DO:
✓ Target keywords with 500-5000 monthly volume
✓ Choose keywords with difficulty < 40
✓ Include primary keyword in title, first 100 words, meta
✓ Use secondary keywords naturally
✓ Target long-tail keywords (lower competition)

DON'T:
✗ Stuff keywords (unnatural placement)
✗ Use keywords that don't match content
✗ Ignore user search intent
✗ Only focus on main keyword
```

### Content Best Practices
```
STRUCTURE:
✓ 1500+ words for competitive topics
✓ 5-10 H2/H3 headings
✓ 1 image per 300-400 words
✓ Short paragraphs (2-3 sentences max)
✓ Bulleted lists for scanability

OPTIMIZATION:
✓ Primary keyword in H1 title
✓ Secondary keywords in H2/H3
✓ Internal links (3-5 minimum)
✓ External links to authoritative sources
✓ Alt text for all images
✓ Meta description (150-160 chars)

ENGAGEMENT:
✓ Write for humans, optimize for search engines
✓ Include examples and case studies
✓ Add actionable tips
✓ Use data and statistics
✓ Keep sentences simple and clear
```

### Technical SEO
```
✓ Proper heading hierarchy (H1 → H2 → H3)
✓ Mobile-friendly format
✓ Fast loading time
✓ Clear URL structure (slug)
✓ Structured data (schema markup)
✓ Open Graph tags (social sharing)
✓ Canonical URL (duplicate content)
✓ FAQs structured data
```
